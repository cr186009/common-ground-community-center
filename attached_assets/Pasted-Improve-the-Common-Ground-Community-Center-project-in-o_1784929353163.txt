==================================================
1. PEXELS FALLBACK IMAGE DATA
==================================================

Update prisma/schema.prisma in the Event model.

Add nullable fields:

imageSource       String?
imageCredit       String?
imageCreditUrl    String?
imageAlt          String?
imageIsFallback   Boolean @default(false)

Keep the existing imageUrl field.

The priority for event images must be:

1. Real image scraped from the event source
2. Manually assigned image
3. Persisted Pexels fallback image
4. Existing local generic placeholder

Never replace a real scraped or manually assigned image automatically.

Do not call Pexels every time a public page renders. Search Pexels only from an explicit Admin action and save the selected result in the database.

==================================================
2. PEXELS SERVICE
==================================================

Create:

src/server/pexels.ts

Use the Pexels REST API directly with fetch. Do not add an unnecessary npm package.

Read the API key only from:

process.env.PEXELS_API_KEY

Never expose the key to client-side code.

Implement:

- searchPexelsPhotos(query, options)
- findFallbackImageForEvent(event)
- assignFallbackImageToEvent(eventId)
- assignFallbackImagesToMissingEvents(options)

Use:

GET https://api.pexels.com/v1/search

Send the API key in the Authorization header.

Use landscape orientation and request approximately 15 results.

Choose a result randomly from the returned results, but persist the choice so the image does not change on every page load.

Save:

- photo.src.landscape or photo.src.large as imageUrl
- "Pexels" as imageSource
- photographer name as imageCredit
- photo.url as imageCreditUrl
- photo.alt as imageAlt
- true as imageIsFallback

Build a useful search query based on event category and title.

Suggested category queries:

FAMILY:
"community family festival"

MUSIC:
"outdoor live music community"

FOOD_DRINK:
"community food festival"

FESTIVAL:
"community street festival"

PARKS_RECREATION:
"community park outdoor recreation"

SPORTS:
"community sports event"

LIBRARY:
"community library books"

SCHOOL:
"school community event"

VOLUNTEER:
"community volunteers"

GOVERNMENT_MEETING:
"community town hall meeting"

TRIVIA:
"friends trivia night"

KARAOKE:
"community karaoke music"

OTHER:
"local community gathering"

Do not send private information, database IDs, addresses, or full descriptions to Pexels.

Handle these cases gracefully:

- PEXELS_API_KEY is missing
- no photographs are returned
- Pexels returns an error
- rate limit is reached
- an event already has a non-fallback image

A failed Pexels request must not break the Admin page or public Events page.
